# Руководство по установке оптимизации производительности

## Что было улучшено

### 1. Real-time обновления ленты (вместо polling)
**Было:** Каждые 30 секунд приложение опрашивало Firebase для получения новых scrobbles
**Стало:** Мгновенные обновления через Firebase real-time subscriptions (WebSocket)

**Преимущества:**
- Обновления появляются мгновенно (без задержки в 30 секунд)
- Меньше нагрузки на Firebase (оплата по количеству чтений)
- Лучший UX - пользователь видит изменения сразу

### 2. Адаптивный polling для Spotify API
**Было:** Фиксированный интервал 5 секунд независимо от состояния
**Стало:** Умный адаптивный интервал:
- **10 секунд** - когда играет музыка
- **30 секунд** - когда музыка на паузе или остановлена
- **60+ секунд** - после ошибки (с экспоненциальным увеличением)

**Преимущества:**
- Снижение количества запросов к Spotify API на 50-80%
- Меньше шансов попасть в rate limit
- Экономия батареи на мобильных устройствах

### 3. Exponential backoff при ошибках
**Было:** При ошибке продолжали опрашивать каждые 5 секунд
**Стало:** При повторяющихся ошибках задержка увеличивается: 60s → 120s → 240s → максимум 5 минут

**Преимущества:**
- Не создаём лишнюю нагрузку при проблемах с Spotify API
- Автоматическое восстановление после временных сбоев
- Соблюдение best practices для работы с API

### 4. Обработка rate limit (429 responses)
**Было:** Игнорировали статус 429 от Spotify
**Стало:** Корректная обработка с использованием заголовка `Retry-After`

**Преимущества:**
- Соблюдение лимитов Spotify API
- Предотвращение временной блокировки аккаунта
- Плавная деградация сервиса при проблемах

## Как проверить что всё работает

### Проверка 1: Real-time обновления ленты
1. Откройте приложение в двух вкладках браузера
2. В одной вкладке включите музыку в Spotify
3. Во второй вкладке следите за лентой "Все" или "Мои"
4. **Ожидаемый результат:** Новый scrobble должен появиться **мгновенно**, без задержки в 30 секунд

### Проверка 2: Адаптивный polling Spotify
1. Откройте Developer Tools (F12) → Console
2. Включите музыку в Spotify
3. Следите за сообщениями в консоли
4. **Ожидаемый результат:** Опросы происходят каждые ~10 секунд
5. Поставьте музыку на паузу
6. **Ожидаемый результат:** Интервал между опросами увеличится до ~30 секунд

### Проверка 3: Exponential backoff
1. Откройте Developer Tools (F12) → Network
2. Временно отключите интернет или заблокируйте Spotify API
3. Следите за консолью
4. **Ожидаемый результат:** После первой ошибки подождёт 60с, после второй 120с, и так далее

### Проверка 4: Отсутствие ошибок
1. Откройте Developer Tools (F12) → Console
2. Используйте приложение как обычно
3. **Ожидаемый результат:** Нет красных ошибок связанных с Firebase или Spotify API

## Технические детали изменений

### Изменённые файлы:

1. **src/services/firebase.ts**
   - Добавлены функции `subscribeToRecentScrobbles()`, `subscribeToUserScrobbles()`, `subscribeToFollowingScrobbles()`
   - Используют Firebase `onSnapshot()` для real-time обновлений

2. **src/hooks/useScrobbler.ts**
   - Реализован адаптивный polling с тремя интервалами (10s/30s/60s)
   - Добавлен exponential backoff с максимальной задержкой 5 минут
   - Добавлено отслеживание последовательных ошибок

3. **src/services/spotify.ts**
   - Добавлена обработка HTTP 429 (Too Many Requests)
   - Извлечение заголовка `Retry-After` и генерация соответствующей ошибки

4. **src/pages/FeedPage.tsx**
   - Заменён polling на real-time subscriptions
   - Автоматическая отписка при размонтировании компонента
   - Обработка обновлений через callback `handleScrobblesUpdate`
